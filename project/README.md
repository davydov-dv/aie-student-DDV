## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - `01_eda_wifi_traffic.ipynb` — разведочный анализ данных (EDA);
  - `02_feature_engineering_and_baseline.ipynb` — создание признаков и baseline-модели;
  - `03_model_comparison.ipynb` — обучение и сравнение ML-моделей.
- `src/` – основной код проекта:
  - `src/data/preprocessor.py` — предобработка данных (кодирование категориальных признаков);
  - `src/models/predictor.py` — загрузка модели и инференс;
  - `src/service/app.py` — FastAPI-сервис с эндпоинтами `/health` и `/predict`;
  - `src/config.py` — централизованная конфигурация.
- `data/` – демонстрационные/учебные данные:
  - `data/raw/UNSW_NB15/` — открытый датасет UNSW-NB15;
  - `data/processed/` — обработанные данные (признаки и таргеты);
  - `data/data_description.md` — подробное описание данных.
- `configs/` – конфигурационные файлы:
  - `.env.example` — шаблон переменных окружения;
  - `.env` — рабочая конфигурация (не в Git).
- `tests/` – тесты (юнит-тесты API, предобработки и модели):
  - `test_api.py`, `test_preprocessor.py`, `test_predictor.py`.
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов:
  - `lgbm_anomaly_model.pkl` — финальная модель LightGBM;
  - `label_encoders.pkl` — энкодеры категориальных признаков;
  - `visualizations/` — графики EDA и сравнения моделей.

## 5. Данные

## 5. Данные

Для обучения и тестирования используется открытый академический датасет **UNSW-NB15** (Australian Centre for Cyber Security, University of New South Wales).

- **Источник:** [Kaggle UNSW-NB15](https://www.kaggle.com/datasets/mrwellsdavid/unsw-nb15)
- **Размер:** 257,673 сетевых потоков (175,341 train + 82,332 test)
- **Признаки:** 42 числовых и категориальных метрики (байты, пакеты, длительность, протокол и т.д.)
- **Классы:** Normal + 9 типов атак (DoS, Exploits, Fuzzers, Reconnaissance, Backdoors, Shellcode, Worms, Analysis, Generic)
- **Расположение:** `data/raw/UNSW_NB15/`

Подробное описание данных — в [`data/data_description.md`](data/data_description.md).

**Важно:** датасет не содержит персональных данных, все IP-адреса синтетические. Использование в академических целях свободно.

## 8. Ограничения и дальнейшая работа

> 1. Текущие ограничения
> - Синтетические временные метки: в датасете UNSW-NB15 отсутствуют реальные timestamp'ы, поэтому агрегация во временные окна выполнена искусственно. Мы работаем на уровне сетевых потоков (flows), а не временных рядов.
> - Только бинарная классификация: модель определяет «норма/атака», но не классифицирует тип атаки (хотя признаки для этого есть).
> - Одна финальная модель: в продакшне используется только LightGBM, без ансамблей или A/B-тестирования.
> - Нет авторизации и rate limiting в API.
> 2. Направления развития
> В дальнейшем можно добавить:
> - Многоклассовая классификация: предсказывать конкретный тип атаки (DoS, Exploits и т.д.).
> - Реальные данные: подключить сервис к реальному роутеру (OpenWrt/MikroTik) и собирать трафик в реальном времени.
> - Мониторинг: добавить Prometheus + Grafana для метрик API (latency, throughput, ошибки).
> - Версионирование моделей: использовать MLflow или DVC для управления версиями моделей.
> - CI/CD: настроить GitHub Actions для автоматического тестирования и деплоя.
> - Streaming-инференс: обрабатывать потоки данных через Kafka или RabbitMQ.
